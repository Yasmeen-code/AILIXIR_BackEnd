# v2.2 - Hybrid Multi-Fingerprint Engine with Advanced Ranking (May 31, 2026)

**Status**: ✅ **RELEASED** - Enterprise-grade chemical similarity search with multi-dimensional ranking

## Major Innovation: 4-Fingerprint Hybrid Reranking Pipeline

A comprehensive redesign from single Morgan fingerprint (v2.1) to **hybrid multi-fingerprint architecture** with advanced ranking algorithms.

## Architecture Layers

### **Layer 1: FAISS Binary Retrieval** 🚀
- **Purpose**: Ultra-fast broad retrieval over Morgan fingerprints
- **Speed**: <100ms for 1M compounds
- **Output**: Top 200 candidate molecules (wide net)
- **Index**: `IndexBinaryFlat` with Hamming distance metric

### **Layer 2: Multi-Fingerprint Reranking** 🔬
Four complementary fingerprint types for comprehensive molecular characterization:

| Fingerprint | Type | Captures | Weight |
|-------------|------|----------|--------|
| **Morgan** | Circular topology | Structural neighborhoods | **50%** |
| **MACCS Keys** | Functional groups | Substructure patterns | **20%** |
| **Atom Pairs** | Long-range geometry | Atomic relationships | **20%** |
| **Topological Torsions** | 4-atom paths | Conformational features | **10%** |

- **Computation**: Exact Tanimoto similarity on all 4 fingerprints
- **Fusion**: Weighted average (50% Morgan + 20% MACCS + 20% Atom Pairs + 10% Torsions)
- **Result**: Hybrid score incorporating multiple chemical perspectives

### **Layer 3: Similarity Calibration** 📊
- **Step 1**: Compute Z-score normalization of hybrid scores
- **Step 2**: Apply logistic sigmoid transformation → probability distribution [0.0, 1.0]
- **Purpose**: Normalize scores and handle outliers mathematically
- **Result**: Calibrated score with statistical soundness

### **Layer 4: Maximal Marginal Relevance (MMR)** 🌈
Diversity control to eliminate redundancy in results:
- **Formula**: `MMR = λ × relevance - (1-λ) × max_similarity_to_selected`
- **Parameter**: `lambda_param` (default: 0.6)
  - `λ = 1.0`: Pure relevance (no diversity)
  - `λ = 0.5`: Balanced relevance + diversity
  - `λ = 0.0`: Maximum diversity (minimum redundancy)
- **Algorithm**:
  1. Pick highest-scored compound first
  2. For each remaining candidate, compute MMR with already-selected items
  3. Greedily select compound with highest MMR value
  4. Repeat until k results obtained

## Key Improvements Over v2.1

### 1️⃣ **Multi-Dimensional Assessment**
- v2.1: Only Morgan fingerprints considered
- v2.2: 4 complementary fingerprint types for holistic molecular profiling
- **Benefit**: Captures structural, functional, spatial, and conformational aspects

### 2️⃣ **Redundancy Elimination**
- v2.1: Multiple similar compounds in top-k results
- v2.2: MMR algorithm ensures diverse chemical space coverage
- **Benefit**: Users get varied alternatives, not duplicates

### 3️⃣ **Statistical Calibration**
- v2.1: Raw hybrid scores (arbitrary scale)
- v2.2: Z-score normalized + sigmoid transformation → [0.0, 1.0] probability space
- **Benefit**: Scores are mathematically interpretable and comparable

### 4️⃣ **Fine-Tuned Weighting**
- v2.1: Equal weight (25% each fingerprint type)
- v2.2: Optimized weights based on chemical relevance
  - Morgan (50%): Most important for structure
  - MACCS (20%): Functional group detection
  - Atom Pairs (20%): Spatial relationships
  - Torsions (10%): Conformational nuance
- **Benefit**: Results reflect chemical importance hierarchy

## Configuration Parameters

```python
engine.search(
    query_smiles="c1ccccc1",  # Benzene
    k=3,                      # Number of results
    lambda_param=0.6          # MMR balance (0.0-1.0)
)
```

- **k**: Final result count (1-20 typical)
- **lambda_param**: Controls relevance vs. diversity trade-off
- **Candidate pool**: Automatically scaled (50-200 molecules from FAISS)

## Implementation Details

### Engine Class Structure
```python
ChemicalSearchEngine(bit_size=2048):
  - _mol_to_all_fingerprints()      # Generate 4 fingerprint types
  - add_compounds(smiles_list)      # Build hybrid index
  - search(query_smiles, k, lambda) # Execute full pipeline
  - save_index(filepath)            # Persist all fingerprints
  - load_index(filepath)            # Restore infrastructure
```

### Fingerprint Storage
- Morgan, MACCS, Atom Pairs, Torsions: Stored as RDKit ExplicitBitVect
- FAISS format: Packed uint8 matrix (N, 256) for binary indexing
- Metadata: Compound info (CID, name, molecular weight, SMILES)
- Persistence: Pickle + separate FAISS binary file
